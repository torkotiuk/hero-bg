Form 3. Guide for label moving up.

  1. Make some div with input, label & span (as a icon)
    • <div class="form3__field">
      <input class="form3__input" type="text" id="form3__name" />
      < label class="form3__label" for="form3__name">Name</label>
      <span class="form3__span"></span>
      <div>
    • .form3__field {position: relative;}
    • .form3__label {
      position: absolute;
      top: 0;
      left: 0;
      transition: transform 250ms linear;
      }
  2. Add selector for moving label
    .form3__input:focus + .form3__label {transform: translateY(-30px);}
  3. After typing text in INPUT live label in place where it was after point 2 by adding selector
    • .form3__input:not(:placeholder-shown) + .form3__label {transform: translateY(-30px);}
    • add empty placeholder=" " for markup
      <input class="form3__input" type="text" id="form3__name" placeholder=" " />

Form 5. Custom checkbox.

Guide:

  1. Make markup:
    <label>
    <input class="checkbox" type="checkbox" />
    <span class="icon"></span>
    HTML
    </label>
  2. Make appearance: none; for all browsers by service autoprefixer and take out checkbox anywhere by position: absolute, like
    .checkbox {
    -webkit-appearance: none;
    -moz-appearance: none;
    appearance: none;
    position: absolute;
    }
  3. Make some icon, like
    .icon {
    content: '';
    display: inline-block;
    width: 30px;
    height: 30px;
    background-color: grey;
    }
  4. Then change smth. in icon
    .checkbox:checked + .icon {
    background-color: green;
    }
  5. For making better clicking in label make it inline-block/block
    label {
    outline: 1px solid black;
    display: inline-block;
    }

Info:

appearance in caniuse.com - don't support all browsers, so got to autoprefixer and make your checkbox with prefix-es for all browsers, like take
  • .checkbox { appearance: none; }

Some form info:

  • If we have couple forms in one page, we have to set different id's for inputs, but we could set the same name-s in different form in one page, f.e.
    • form1, input name=user id="form1-user"
    • form2, input name=user id="form2-user"
  • In a group of checkbox possibly use different names, but more convenient for js & backend of using one name for one group of checkbox-es
  • HOOK - Group some elements in form with div role="" (for increased semantic ) - webinar 00:26:00
  • use one checked attribute for radio buttons & many checked for checkbox buttons
  • use attribute autofocus
  • use attribute autocomplete as for all form and for any element, f.e.
    • form autocomplete="off"
    • input name="username" autocomplete="on"
    Also there are specific autocomplete values (name, email, country et.c.)
  • Don't style checkbox in usual css, b/c it looks in different browsers really bad
  • Use autoprefixer for making one simple css style for all browsers
