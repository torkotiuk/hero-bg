Case when we made margins for every element, but for container we made minus margin

calc((100% - amount of margins * 5px) / amount of elements); f.e. (explained for tab & pc views) in First section we got 10px. margins between elements, so we have 5px. margin for every element,

  • for tablet view we would have formula - calc((100% - 4 * 5px) / 2), where we have 4 margins for 2 elements
  • for pc view we would have formula - calc((100% - 6 * 5px) / 3), where we have 6 margins for 3 elements

Burger menu guide

P.S. Blocks with color for accessibility reasons, for start enough add <svg aria-label="Mobile menu switch"></svg> , so point 5, 7 should add after everything

  1. Add necessary icons to sprite.
  2. Create simplest markup for burger button
    • <button type="button" class="menu-button">
      <svg width="40" height="40">
      <use class="icon-menu" href="./images/sprite.svg#icon-menu"></use>
      <use class="icon-close" href="./images/sprite.svg#icon-close"></use>
      </svg>
      </button>
    • Also don't forget about accessibility
      <svg aria-label="Mobile menu switch">
  3. Styles for button
    • Add basically styles
      .menu-button {
      display: inline-flex;
      margin: 0;
      padding: 0;
      border: none;
      background-color: transparent;
      }
    • Hide close-icon
      .menu-button .icon-close {
      display: none;
      }
    • When additional class "is-open" added
      • Show close-icon
        .menu-button.is-open .icon-close {
        display: block;
        }
      • Hide "icon-menu"
        .menu-button.is-open .icon-menu {
        display: none;
        }
    • Add data attribute for button & js for icons toggling
      • <body>
        ...
        <button type="button" class="menu-button" data-menu-button>
        <svg width="40" height="40" aria-label="Mobile menu switch">
        <use class="icon-menu" href="./images/sprite.svg#icon-menu"></use>
        <use class="icon-close" href="./images/sprite.svg#icon-close"></use>
        </svg>
        </button>
        ...
        (() => {
        const menuBtnRef = document.querySelector('[data-menu-button]');
        menuBtnRef.addEventListener('click', () => {
        menuBtnRef.classList.toggle('is-open');
        });
        })();
        </body>
  5. Additional accessibility point, add aria-expanded attribute, that READERS tell user about menu opened or closed
    ...
    <button type="button" class="menu-button" aria-expanded="false" data-menu-button>
    <svg width="40" height="40" aria-label="Mobile menu switch">
    <use class="icon-menu" href="./images/sprite.svg#icon-menu"></use>
    <use class="icon-close" href="./images/sprite.svg#icon-close"></use>
    </svg>
    </button>
    ...
    (() => {
    const menuBtnRef = document.querySelector('[data-menu-button]');
    menuBtnRef.addEventListener('click', () => {
    const expanded = menuBtnRef.getAttribute('aria-expanded') === 'true' || false;
    menuBtnRef.classList.toggle('is-open');
    menuBtnRef.setAttribute('aria-expanded', !expanded);
    });
    })();
  6. Add some markup of menu content after out button & fulfill js
    • <div class="mob-menu-container" data-menu>Menu content</div>
    • (() => {
      const menuBtnRef = document.querySelector('[data-menu-button]');
      const mobileMenuRef = document.querySelector('[data-menu]');
      menuBtnRef.addEventListener('click', () => {
      const expanded = menuBtnRef.getAttribute('aria-expanded') === 'true' || false;
      menuBtnRef.classList.toggle('is-open');
      menuBtnRef.setAttribute('aria-expanded', !expanded);
      mobileMenuRef.classList.toggle('is-open');
      });
      })();
    • Also add some basically styles for mob-menu-container, that positioned absolutely for header, so make header: relative;
      .mob-menu-container {
      position: absolute;
      top: 100%;
      left: 0;
      width: 100%;
      height: 250px;

      display: none;

      background-color: palegoldenrod;
      padding: 20px;
      text-align: center;
      font-size: 20px;

      &.is-open {
      display: block;
      }
      }
  7. Connect menu with button for accessibility
    • <div class="mob-menu-container" id="menu-container" data-menu> Menu content </div>
    • <button
      type="button"
      class="menu-button"
      aria-expanded="false"
      aria-controls="menu-container"
      data-menu-button
      >
      ...
      </button>
