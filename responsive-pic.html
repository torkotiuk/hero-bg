Responsive images

This pic is responsive till the end of the responsive container 480px.

Guide

  • Make in every project for tag img
    • img { display: block; max-width: 100%; height: auto; }

      P.S. max-with - use for pic-s not more than pic-s size, means if we made width we will get pic-s in responsive blocks bigger than real size.

  • Go to mydevice.io for measure of your CSS pixel-ratio screen. For screens with this value much than 1, we have to prepare bigger pic-s.
  • Approach 1. with fixed <img width="">, usually in adaptive markup Prepare pic-s for different screen sizes, means export from f.e. Figma and markup it like. (How to prepare bigger (for retina) pic-s with Figma.)

    So browser downloads pic for selected screen with CSS pixel-ratio

    • <img
      srcset="
      ./assets/images/cat-320.jpg 1x,
      ./assets/images/cat-640.jpg 2x,
      ./assets/images/cat-960.jpg 3x
      "
      src="./assets/images/cat-320.jpg"
      alt="Cat"
      width="320"
      />
  • Approach 2. Browser takes nearest for screen quality pic. <img
    srcset="
    ./assets/images/cat-320.jpg 320w,
    ./assets/images/cat-640.jpg 640w,
    ./assets/images/cat-960.jpg 960w
    ./assets/images/cat-1280.jpg 1280w
    "
    src="./assets/images/cat-320.jpg"
    alt="Cat"
    sizes="(min-width: 800px) 33vw, (min-width: 500px) 50vw, 100vw"
    />

    Attribute sizes="" helps us optimize of pic-s selecting, f.e. (min-width: 800px) 50vw - means from 800px we will show pic not much than 50% of view port. More explanation in webinar.

  • Try use lazy attribute in markups
    • <img loading="lazy"> - images don't paint when they aren't in screen
  • Good practice of page downloading is 1,5 - 2 s.
